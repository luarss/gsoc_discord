Hey <@756569592746213508> ,I’ve been thinking about the scaling limit of our bridge. Right now, we’re flattening the eSim subcircuits into a single Verilog module to keep things simple.But if a user tries to build a huge design, OpenROAD is going to hit a wall. If we flatten everything, the Global Router loses the 'hints' about which components should stay together, and we'll get impossible routing congestion. But if we keep the hierarchy, the net names in the final GDS will be totally different from the names in the eSim schematic (because of the X1.inst2.net prefixing).

How do we handle the back-annotation?If OpenROAD moves or renames a net to fix a timing path, how are we supposed to map that 'physical' reality back to the 'logical' eSim schematic so the user can actually simulate their post-layout results? Or are we just stuck with 'Flattened' designs forever?
