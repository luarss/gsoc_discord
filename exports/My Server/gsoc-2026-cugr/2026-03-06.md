### 06:03 - @Soumyadip

Hi <@1478370910749855888>, Just pushed a fix for **PR #9642 **(https://github.com/The-OpenROAD-Project/OpenROAD/pull/9642). 
The root cause was a missing type definition for GPoint3D in GridGraph.h. Since GridGraph.h is included transitively through `GRNet.h → CUGR.cpp`, the compiler couldn't resolve GPoint3D at the point where findPointIndex was declared  so, even though GeoTypes.h was available in `GridGraph.cpp`. The declaration and the definition were in different translation units with different include chains.
 Adding `#include `"GeoTypes.h" directly into GridGraph.h so that GPoint3D is always resolved regardless of which file includes GridGraph.h first. This follows the principle that a header should be self-contained  and it should never rely on the includer to have already pulled in its dependencies.
DCO is also fixed with -s on all commits.
