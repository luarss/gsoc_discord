### 08:09 - @Soumyadip

Hey <@756569592746213508>  <@1479005816982868039>  ,I've opened PR #467 as a starting point for the eSim-ORFS integration here -https://github.com/FOSSEE/eSim/pull/467 . The approach is to first parse the `.cir.out` netlist exported from eSim's schematic editor, then validate all components against the target PDK standard cells to flag any unmappable primitives before they reach ORFS. Once validation passes, the script auto-generates the ORFS config.mk so the user doesn't have to manually configure anything, and then triggers the ORFS flow automatically. The validator lives outside the GUI as a preprocessing flow script  and keeping it modular and easier to test independently, as we discussed earlier.Would love your thoughts on this approach before I build out the full implementation.
