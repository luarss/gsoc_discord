### 20:29 - @Kartik

Hi Jack,
Been digging into the openroad-mcp codebase making good progress toward the Docker deployment. Hit a few questions before I start writing the Dockerfile and CI workflow, didn't want to go too far down the wrong path:

1. PTY thing in Docker -   the server talks to OpenROAD over PTY, which needs a real terminal. Containers don't have one without docker run -it, which is probably why make test-interactive gets skipped in CI. For the prod image, is there a TCP/socket transport mode on your radar, or should the initial Docker setup just require -it and document the limitation? TCP would be much cleaner long-term.

2. Base image - build OpenROAD from scratch inside the Dockerfile, or just use openroad/flow-ubuntu22.04-dev and layer the MCP server on top? The second option gets build time from ~90 min down to ~2 min which actually makes CI usable. Leaning toward that but wanted to check it fits what you had in mind.

3. GHCR namespace - do I push to ghcr.io/luarss/openroad-mcp or is there a plan to move it under the OpenROAD org at some point? Don't want to wire the workflow to the wrong place and have it break later.
