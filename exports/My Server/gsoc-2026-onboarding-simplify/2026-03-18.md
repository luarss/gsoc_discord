Hi <@1059677402315370658>  , I’ve been diving into the infrastructure side for the Onboarding Simplification project. I noticed that when users build from source locally, missing a dependency (like bison or flex) causes the build to fail deep into the CMake process with cryptic errors.To improve the DX for beginners, I just opened a PR (https://github.com/The-OpenROAD-Project/OpenROAD/pull/9805) that adds a "Pre-flight System Check" to Build.sh. It fails fast (in ~3 seconds) and tells the user exactly which tool is missing and instructs them to run DependencyInstaller.sh before CMake even attempts to configure.

### 15:07 - @Soumyadip (Divinesoumyadip-git)

<@1059677402315370658>  I also opened a second, separate PR (https://github.com/The-OpenROAD-Project/OpenROAD/pull/9806) for the infrastructure side. I noticed the main Dockerfile was carrying around a lot of redundant apt cache metadata.I’ve optimized the dev stage by purging /var/lib/apt/lists/ within the same RUN layer as the dependency installation. This significantly reduces the base image footprint, which carries through to the builder and final stages. It should result in faster CI/CD pulls and a leaner environment for anyone using the Docker-based onboarding flow.
